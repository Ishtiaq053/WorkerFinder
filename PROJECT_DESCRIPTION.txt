WORKERFINDER PROJECT - COMPREHENSIVE DESCRIPTION
=====================================================

WorkerFinder is a full-stack web application that serves as a labor marketplace platform 
connecting customers with skilled workers. The project is built using React (frontend) 
and Node.js/Express (backend) with file-based data storage.

PROJECT STRUCTURE OVERVIEW
===========================

Frontend Structure (WorkerFinder/frontend/):
- Framework: React with Vite bundler
- Routing: React Router DOM
- Styling: Bootstrap 5 + Custom CSS
- State Management: React Context API for authentication
- Build Tool: Vite with proxy configuration for API calls

Backend Structure (WorkerFinder/backend/):
- Framework: Express.js
- Authentication: JWT tokens
- File Upload: Multer middleware
- Data Storage: File-based JSON storage (no database)
- CORS: Enabled for cross-origin requests

DETAILED FILE STRUCTURE & FUNCTIONALITY
========================================

Core Application Files
----------------------

Frontend Entry Point:
- main.jsx: Application bootstrap that wraps the app with React Router and Auth Context
- App.jsx: Main router configuration defining all public and protected routes
- index.html: HTML template with Bootstrap CDN links

Backend Entry Point:
- server.js: Express server setup with middleware configuration, route mounting, and server startup

AUTHENTICATION SYSTEM
======================

Context & State Management:
- AuthContext.jsx: Global authentication state using React Context, manages user session persistence in localStorage

Authentication Pages:
- Login.jsx: User login with form validation and role-based redirects
- Signup.jsx: Registration with role selection (customer/worker) and skill selection for workers

Backend Authentication:
- authRoutes.js: Authentication endpoints (signup, login, logout, profile)
- authController.js: Authentication logic and JWT token management
- auth.js: JWT verification middleware for protected routes

USER ROLES & DASHBOARDS
========================

Customer Dashboard:
- UserDashboard.jsx: 
  * Overview: Statistics and job summary
  * Post Jobs: Create new job postings with title, description, budget, category
  * My Jobs: View posted jobs, manage status (active/completed/cancelled)
  * Applications: Review worker applications, accept/reject candidates
  * Profile Management: Edit personal information
  * Contact Support: Embedded contact form

Worker Dashboard:
- WorkerDashboard.jsx:
  * Overview: Profile status and application statistics  
  * Browse Jobs: View available jobs (only if worker is approved)
  * Apply for Jobs: Submit applications with cover notes
  * My Applications: Track application status (pending/accepted/rejected)
  * Profile Management: Complete worker profile with skills and location
  * Contact Support: Help and support access

Admin Dashboard:
- AdminDashboard.jsx:
  * Platform Stats: Total users, workers, jobs, applications
  * Worker Management: Approve/reject pending worker registrations
  * Job Management: View all jobs, delete inappropriate postings
  * System Messages: Handle user inquiries and support messages

- jobRoutes.js: Job CRUD operations with role-based access

Application System:
- applicationRoutes.js: Worker applications to jobs

- adminRoutes.js: Administrative operations

DATA FLOW ARCHITECTURE
=======================

1. Authentication Flow: Login → JWT Token → Context State → Protected Routes
2. Job Flow: Customer posts job → Workers browse → Applications submitted → Customer reviews → Job completion
3. Worker Approval: Registration → Admin review → Approval/Rejection → Access to jobs
4. File Upload: File selection → Validation → Upload → Server storage → URL reference

CONCLUSION
==========

This project demonstrates a complete full-stack application with role-based access control, 
file management, and a comprehensive user interface suitable for a professional labor 
marketplace platform. The architecture supports scalability and maintainability while 
providing a seamless user experience across all user roles.
